# Cartographer (Hybrid)

Two-tier codebase mapping system. Tier 1 for high-level orientation, Tier 2 for module-level detail.

## When to Use Each Tier

| Trigger | Tier |
|---------|------|
| First mapping of a new repo | Both (Tier 1 + Tier 2 init) |
| Major structural change (new modules, renames) | Tier 1 (regenerate CODEBASE_MAP.md) |
| After PR merge (routine) | Tier 2 only (update affected codemap.md) |
| Agent needs project overview | Read Tier 1 (`docs/CODEBASE_MAP.md`) |
| Agent needs module detail | Read Tier 2 (folder's `codemap.md`) |

## Full /cartographer Flow

When user invokes `/cartographer`, run both tiers:

1. **Tier 1** — Generate or update `docs/CODEBASE_MAP.md`
2. **Tier 2 init** — Run `cartographer.py init` if `.slim/cartography.json` doesn't exist
3. **Tier 2 detect** — Run `cartographer.py changes` to find affected folders
4. **Tier 2 fill** — Dispatch haiku explorers to fill affected `codemap.md` files
5. **Tier 2 save** — Run `cartographer.py update` to record new hashes

## Tier 1: docs/CODEBASE_MAP.md

High-level architecture atlas. Generated by parallel Sonnet subagents, updated rarely.

**CRITICAL: Opus orchestrates, Sonnet reads.** Never have Opus read codebase files directly. Always delegate file reading to Sonnet subagents.

### Step 1: Check for Existing Map

Check if `docs/CODEBASE_MAP.md` already exists:

**If it exists:**
1. Read the `last_mapped` timestamp from the map's frontmatter
2. Run `git log --oneline --since="<last_mapped>"` to check for changes
3. If significant changes detected, proceed to update mode
4. If no changes, inform user the map is current

**If it does not exist:** Proceed to full mapping.

### Step 2: Scan the Codebase

Run the scanner script to get file tree with token counts:

```bash
# Option 1: UV (preferred)
uv run ${CLAUDE_PLUGIN_ROOT}/scripts/scan-codebase.py . --format json

# Option 2: python3 fallback
python3 ${CLAUDE_PLUGIN_ROOT}/scripts/scan-codebase.py . --format json
```

The output provides: file tree with token counts, total token budget, skipped files.

### Step 3: Plan Subagent Assignments

**Token budget per subagent:** ~150,000 tokens (safe margin under 200k context).

Group files by directory/module, balance token counts. For small codebases (<100k tokens), use a single Sonnet subagent.

### Step 4: Spawn Sonnet Subagents in Parallel

Use the Agent tool with `subagent_type: "Explore"` and `model: "sonnet"` for each group.

**CRITICAL: Spawn all subagents in a SINGLE message.**

Each subagent prompt should request:
- Purpose of each file/module
- Key exports and public APIs
- Dependencies and dependents
- Patterns and conventions
- Gotchas or non-obvious behavior

### Step 5: Synthesize Reports

Merge all subagent reports. Deduplicate, identify cross-cutting concerns, build architecture diagram, extract navigation paths.

### Step 6: Fixer Writes CODEBASE_MAP.md

Dispatch **Fixer** (haiku) to write the synthesized output. Sonnet subagents analyze — Fixer writes the file.

**Get the actual timestamp first:**
```bash
date -u +"%Y-%m-%dT%H:%M:%SZ"
```

Create `docs/CODEBASE_MAP.md`:

```markdown
---
last_mapped: YYYY-MM-DDTHH:MM:SSZ
total_files: N
total_tokens: N
---

# Codebase Map

> Auto-generated by Cartographer. Last mapped: [date]

## System Overview
[Mermaid diagram showing high-level architecture]

## Directory Structure
[Tree with purpose annotations]

## Module Guide
### [Module Name]
**Purpose**: [description]
**Entry point**: [file]
**Key files**: [table with file, purpose, tokens]
**Exports**: [key APIs]
**Dependencies**: [what it needs]
**Dependents**: [what needs it]

## Data Flow
[Mermaid sequence diagrams for key flows]

## Conventions
[Naming, patterns, style]

## Gotchas
[Non-obvious behaviors, warnings]

## Navigation Guide
**To add a new feature**: [files to touch]
**To modify a hook**: [files to touch]
```

### Step 7: Update CLAUDE.md

Add or update codebase summary in CLAUDE.md pointing to the map.

### Tier 1 Update Mode

When updating an existing map:
1. Identify changed files from git
2. Spawn Sonnet subagents only for changed modules (read + analyze)
3. **Fixer** (haiku) merges new analysis with existing map and writes the file
4. **Fixer** updates `last_mapped` timestamp
5. Preserve unchanged sections

## Tier 2: Per-Folder codemap.md

Module-level detail. Managed by `cartographer.py` with MD5 change detection. Cheap and incremental.

### Initialize (first time only)

Analyze the repo and choose include/exclude patterns for **core code/config only**:
- **Include**: Source code, configs (e.g., `**/*.ts`, `**/*.py`, `**/*.json`)
- **Exclude (MANDATORY)**: Tests, docs, build artifacts, deps

```bash
python3 ${CLAUDE_PLUGIN_ROOT}/scripts/cartographer.py init \
  --root ./ \
  --include "src/**/*.ts" \
  --exclude "**/*.test.ts" --exclude "dist/**" --exclude "node_modules/**"
```

This creates:
- `.slim/cartography.json` — file and folder hashes
- Empty `codemap.md` in all relevant subdirectories

### Detect Changes

```bash
python3 ${CLAUDE_PLUGIN_ROOT}/scripts/cartographer.py changes --root ./
```

Shows: added files, removed files, modified files, affected folders.

### Fill Affected Codemaps

For each affected folder, dispatch a **haiku explorer** to read the folder's files and fill its `codemap.md`:

```markdown
# folder-name/

## Responsibility
What is this folder's job in the system?

## Design
Key patterns, abstractions, architectural decisions.

## Flow
How does data/control flow through this module?

## Integration
How does it connect to other parts of the system?
```

Use precise technical terminology. Be concise — each codemap should be ~200 tokens.

### Save Updated State

```bash
python3 ${CLAUDE_PLUGIN_ROOT}/scripts/cartographer.py update --root ./
```

## Token Budget Reference

| Model | Context Window | Safe Budget per Subagent |
|-------|---------------|-------------------------|
| Sonnet | 200,000 | 150,000 |
| Haiku | 200,000 | 100,000 |

Tier 1: use Sonnet subagents. Tier 2: use Haiku explorers.

## Troubleshooting

**Scanner fails with tiktoken error:**
```bash
pip install tiktoken
# or use uv (auto-installs):
uv run scan-codebase.py .
```

**cartographer.py "No state found":**
Run `cartographer.py init` first — see Tier 2 Initialize section.

**Codebase too large for subagents:**
Increase number of subagents, focus on src/ directories, use `--max-tokens` flag.
